Ringkasan Materi (tanpa Bab/Slide 7–9)

Rangkuman ini mengikuti poin penting dari slide sesi 1.

1) Pendahuluan OOP

  • OOP adalah paradigma pemrograman yang fokus pada objek sebagai unit utama.
  • Objek punya state (data/atribut) dan behavior (perilaku/metode).
  • Tujuan OOP: kode lebih modular, reusable, dan mudah dipelihara.

2) Objek, Kelas, Atribut, Metode

  • Kelas = blueprint/cetakan untuk membuat objek.
  • Objek = instance nyata dari kelas (contoh: mobilSaya dari class Mobil).
  • Atribut = variabel di dalam kelas (misal: warna, model).
  • Metode = fungsi di dalam kelas (misal: start(), stop()).

3) Imperatif vs OOP

  • Imperatif: fokus pada langkah-langkah (function, loop, condition).
  • OOP: fokus pada objek yang saling berinteraksi untuk menyelesaikan masalah.

4) Mengapa OOP di Dart?

  • Dart mendukung OOP secara native (multi-paradigm).
  • Flutter sangat bergantung pada konsep OOP (widget, class, objek).
  • OOP membantu struktur kode saat aplikasi makin kompleks.

5) Sintaks Dasar Kelas di Dart

  • Bentuk umum: class NamaClass { properties; method; }
  • Contoh sederhana: class Car dengan model, color, dan method start(), stop().

6) Membuat Objek dari Kelas

  • Membuat instance: Car myCar = Car();
  • Isi atribut: myCar.model = "Sedan";
  • Panggil method: myCar.start(); myCar.stop();
  • Bisa buat banyak objek dari satu class (misal: anotherCar).

7) Contoh Constructor & Akses Properti

  • Constructor dipakai untuk menginisialisasi properti saat objek dibuat.
  • Contoh konsep: properti privat (pakai _), lalu akses lewat setter/getter.

8) This Keyword

  • this merujuk ke instance saat ini.
  • Berguna saat nama parameter sama dengan nama atribut.
  • Disarankan pola aman: properti non-nullable diinisialisasi lewat constructor.

9) Ringkasan Sesi

  • OOP fokus pada objek & interaksi.
  • Kelas = blueprint, objek = instance.
  • Constructor untuk inisialisasi objek.

10) Quiz & Diskusi

  • Bedakan kelas vs objek, sebutkan keuntungan OOP.
  • Latihan: buat class Hewan (atribut nama, method bersuara()).
  • Diskusi: kapan OOP lebih cocok digunakan.
