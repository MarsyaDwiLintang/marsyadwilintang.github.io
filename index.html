Ringkasan Materi dari Slide (Tanpa Slide 7–9)

Poin penting sesi 1: Pengantar OOP dan dasar kelas di Dart.

1) Pendahuluan OOP

  • OOP adalah paradigma pemrograman yang fokus pada objek sebagai unit utama. :contentReference[oaicite:1]{index=1}
  • Objek punya state (data) dan behavior (perilaku). :contentReference[oaicite:2]{index=2}
  • Tujuan OOP: kode lebih modular, reusable, dan mudah dipelihara. :contentReference[oaicite:3]{index=3}

2) Objek, Kelas, Atribut, Metode

  • Objek = instance nyata dari sebuah kelas. :contentReference[oaicite:4]{index=4}
  • Kelas = blueprint/cetakan untuk membuat objek. :contentReference[oaicite:5]{index=5}
  • Atribut = variabel di dalam kelas (misal: warna, model). :contentReference[oaicite:6]{index=6}
  • Metode = fungsi di dalam kelas (misal: start(), stop()). :contentReference[oaicite:7]{index=7}

3) Imperatif vs OOP

  • Imperatif fokus pada langkah prosedural (function, loop, condition). :contentReference[oaicite:8]{index=8}
  • OOP fokus pada objek yang saling berinteraksi. :contentReference[oaicite:9]{index=9}

4) Mengapa OOP di Dart?

  • Dart bahasa multi-paradigm dan mendukung OOP secara native. :contentReference[oaicite:10]{index=10}
  • Flutter sangat bergantung pada OOP. :contentReference[oaicite:11]{index=11}
  • OOP bikin struktur kode lebih rapi untuk aplikasi kompleks. :contentReference[oaicite:12]{index=12}

5) Sintaks Dasar Kelas di Dart

  • Bentuk umum: class NamaClass { properties; method; }. :contentReference[oaicite:13]{index=13}
  • Contoh class Car dengan atribut dan method start() / stop(). :contentReference[oaicite:14]{index=14}

6) Membuat Objek dari Kelas

  • Membuat instance dari class (contoh: Car myCar = Car();). :contentReference[oaicite:15]{index=15}
  • Bisa buat beberapa objek lalu panggil method-nya. :contentReference[oaicite:16]{index=16}

7) Tipe Constructor (lanjutan sesi)

  • Constructor untuk inisialisasi properti objek. :contentReference[oaicite:17]{index=17}
  • Contoh: default constructor, named constructor, parameterized constructor. :contentReference[oaicite:18]{index=18}

8) This Keyword

  • this merujuk pada instance saat ini, berguna jika parameter sama dengan atribut. :contentReference[oaicite:19]{index=19}
  • Contoh aman: Product(this.nama) (non-nullable diinisialisasi di konstruktor). :contentReference[oaicite:20]{index=20}

9) Ringkasan Sesi & Quiz

  • Ringkasan: OOP fokus pada objek/interaksi; class = blueprint; object = instance; constructor untuk inisialisasi. :contentReference[oaicite:21]{index=21}
  • Quiz: bedakan kelas vs objek, sebutkan keuntungan OOP, buat class Hewan, dan diskusi kapan OOP dipakai. :contentReference[oaicite:22]{index=22}
